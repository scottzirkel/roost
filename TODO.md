## Operational playbook
- **Build:** `./build.sh` from the project root (overlays our `src/roost` into the Ghostty submodule via symlink, applies `patches/build.zig.patch`, runs `zig build roost`) ‚Üí `vendor/ghostty/zig-out/bin/roost`. Zig **0.15.2** at `~/.local/bin/zig`. Edit source in the PARENT `src/roost/` (symlinked in). Gate check: `git -C vendor/ghostty diff --name-only v1.3.1` = only `build.zig`.
- **Toolchain:** if Zig is reinstalled, re-run `scripts/patch-zig-linker.sh` (forces LLVM+LLD around the Arch glibc SFrame relocation ‚Äî Zig issue #31272).
- **Run:** from a git repo `cd <repo> && <‚Ä¶>/zig-out/bin/roost`, or `ROOST_PROJECT=<repo> <‚Ä¶>/roost`. Project dir = `ROOST_PROJECT` ‚Üí cwd. **Never** a positional arg (Ghostty's config parser rejects bare argv).
- **Multi-window caveat (bit us hard while debugging):** each launch writes the same log file and its own `$XDG_RUNTIME_DIR/roost-<pid>.sock`. Before debugging: `pkill -x roost`, launch ONE with a dedicated log, target the right socket (`ls -t $XDG_RUNTIME_DIR/roost-*.sock`).
- **Gate check:** `git -C vendor/ghostty diff --name-only v1.3.1` ‚Üí must be only `build.zig` (our additive step). Our `src/roost`/`main_roost.zig` are untracked symlinks in the submodule (`??`), the canonical copies live in the parent repo.
- **Dev pattern:** delegate each feature to a background `Agent` with a detailed brief (it edits + compiles); then **rebuild + run + verify yourself** ‚Äî agent sandboxes sometimes can't run git/zig and have shipped non-compiling guesses.
- **Reload (dev loop):** edit ‚Üí `./build.sh` ‚Üí **Ctrl+Shift+Q** ‚Üí confirm ‚Üí the window restarts into the freshly-built binary (`win.reload` ‚Üí `onReload`/`doReload`/`reExecSelf`; committed `4f85097`, user-verified). Native code can't hot-swap, so reload **re-execs in place** (`std.posix.execvpeZ` on **`argv[0]`**, reusing our live argv) ‚Äî same PID, NO spawned process, no single-instance forwarding, no quit-race. Layout (roles/splits/**sizes**) is persisted first and restored by the new image; pane processes + scrollback start fresh (unavoidable across a process swap). Confirms first (Cancel default) since it ends everything in the panes. **Gotchas (hard-won, 2026-06-25 ‚Äî both bit us live):**
  - Re-exec MUST use `argv[0]`, not `std.fs.selfExePath`/`/proc/self/exe`: after `./build.sh` replaces the binary, that link resolves to the deleted old inode (`‚Ä¶ (deleted)`) ‚Üí `execve` returns `error.FileNotFound`. `argv[0]` is a plain path that still points at the new file.
  - **Ctrl+Shift+Q** took two fixes: (1) the GTK accelerator must be UPPERCASE `<Ctrl><Shift>Q` (Shift delivers the uppercase keyval; lowercase silently never matches ‚Äî the same gotcha as the other shift-combos); (2) Ghostty binds `ctrl+shift+q ‚Üí quit` by DEFAULT and its terminal surface consumes the chord before our window accelerator fires, so we inject **`--keybind=ctrl+shift+q=unbind`** into our own argv (`injectArgs`, alongside `--class`) to free it. Roost keeps its own quit on Ctrl+Q. (Earlier "spawn + quit" reload was flaky ‚Äî spawn/quit race, single-instance forwarding, and a corrupted flag could strand it on a config-error dialog ‚Äî hence the switch to re-exec.)
- **CLI launcher:** `roost` on PATH is now `scripts/roost-launch` (symlink), a wrapper that `setsid`-detaches the binary, inherits cwd (so `cd proj && roost` still picks the project), logs to `${XDG_STATE_HOME:-~/.local/state}/roost.log`, and closes the launching terminal. The `.desktop` file still points at the binary directly (no terminal there).

## Gotchas (hard-won)
- Code lives in the parent repo's `src/roost/` (+ thin `src/main_roost.zig`) and is symlinked into `vendor/ghostty/src/` at build time so it can relative-import Ghostty internals (it MUST sit inside the Ghostty tree to compile). Only `terminal_pane.zig` names `Surface`; everything else goes through the `TerminalPane` abstraction.
- App id: Ghostty derives it ONLY from `config.class` (valid ‚Üí used verbatim, else default `com.mitchellh.ghostty[-debug]`), loaded from config files + argv inside apprt init. We can't edit Ghostty source (the gate), so we inject `--class` into `std.os.argv` from our entry point. Do NOT set `class` in the user's shared `~/.config/ghostty/config` (would also change real Ghostty).
- zig-gobject: `g_input_stream_read`'s buffer param is typed `*[*]u8` ‚Üí pass `@ptrCast(&buf)` of an `align(8)` buffer. **Shift+digit accelerators don't match** (use Ctrl+Alt or Ctrl+Shift+letter). Structural ops call `tree.syncFocusFromWindow(window)` first ‚Äî `Tree.focused` otherwise only tracks programmatic focus, not mouse clicks.
- gio notifications are silent by design (no system audio).
